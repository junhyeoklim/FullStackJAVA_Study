1. 문자 스트림이 별도로 존재하는 이유 :
 - 바이트 스트림이 존재함에도 불구하고, 문자 스트림이 별도로 필요한 이유는 무엇인가?
	"운영체제의 기본 인코딩 방식으로의 인코딩을 자동화하기 위해서!"

2. Try-with-resource 문 :
 - JAVA 1.7 부터 추가된 try wirh resource 문법을 사용하면 훨씬 간결하게 자원반납 문제를 해결할 수 있게 되었다.,
    try with resource 에서는 try() 내에서 자원할당을 하면 자동으로 close() 메소드가 호출이 된다. 
    때문에 finally 에서 close 해주는 과정을 생략해도 된다.
     ex)
	try (param = new OBject()){
	 ...
	} catch(Exception e) {
	 ...
	}
 * AutoCloseable이 구현 되어 있어야 사용 가능

3. java.io.Serializable : 객체를 파일에 저장 하거나 불러오기 위해 선언 해야 할 것

4. transient : 직렬화 대상에서 제외 시킨다.
	- 복원 과정에서 별도의 입력이 없으면 디폴트 값으로 초기화가 된다.

5. RandomAccessFile : 입력과 출력이 동시에 이뤄질 수 있다.
	- 기술적인 측면으로 보면 RandomAccessFile은 스트림으로 보기 힘들다(이유 : 한 방향으로 흐르는게 아니라서)
