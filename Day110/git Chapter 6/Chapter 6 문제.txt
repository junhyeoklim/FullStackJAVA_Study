다음을 CLI(Command Line Interface)로 실습하자.

1. c:\git-test 에 로컬 저장소를 만들자.
2. 작업 디렉터리 상태를 확인하자.
3. A가 적힌 a.txt 파일을 만들자.
4. 작업 디렉터리 상태를 확인하자.
5. 스테이지에 올리자.
6. first commit 이라는 커밋 메시지로 커밋하자.
7. 저장소의 커밋 목록을 출력하자.
8. a.txt 파일에 B를 추가로 적자.
9. 작업 디렉터리 상태를 출력하자.
10. 스테이지에 올리면서 커밋 메세지를 "second commit"로 하면서
커밋하자.(스테이지에 올리면서 커밋하는 것을 동시에
처리하는 명령어로 처리하자.)
11. a.txt 파일에 C를 추가한 뒤 저장
12.  a.txt 파일을 스테이지로 추가
13. 커밋하고 커밋 제목과 본문을 작성합니다.
제목은 third commit
본문은 This is my third commit
13. 단순한 형태로 커밋 조회
14. 커밋조회(해당 커밋으로 어떤 파일이 어떻게 수정됐는지를 출력하게끔)
15. 그래프 형태로 커밋 조회
16. 모든 브랜치 커밋 조회
17. 현재 커밋에 v1.0.0 태그 추가.
18. 커밋 목록 조회
19. 두 번째 커밋에 v0.0.1 태그추가.
20. 태그 리스트 조회
21. v0.0.1 태그 삭제
22. 태그 조회
23. v1.0.0 태그 삭제
24. 태그 조회
